Pre-Workshop · Local Setup Guide

PerfStack
on your laptop

Install PerfStack locally on your own machine. One script does everything — creates the k3d cluster, builds Docker images, sets up a local registry, and deploys all services. Available at http://localhost with no tunnel, no cloud dependency, and no admin rights needed.

— Docker Desktop k3d K6 + Grafana localhost:80

First-time only

01

Install Required Tools

Only three tools needed. Everything else (Python, Node.js, k6, Grafana, InfluxDB) runs inside the cluster — the deploy script handles it.

All commands run in Terminal.app or iTerm2.

01

Install Homebrew

The macOS package manager. Skip if already installed.

Terminal
bash
$ /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
$ brew --version   # → Homebrew 4.x.x
Apple Silicon (M1/M2/M3/M4): add to PATH after install — echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile && source ~/.zprofile
02

Install Docker Desktop

Provides the container runtime k3d uses. Must be running before deploy.

Terminal
bash
$ brew install --cask docker
# Open Docker Desktop from Applications and complete the setup wizard
# Wait until the whale icon in the menu bar is steady (not animated)
$ docker --version   # → Docker version 26.x.x
Docker Desktop must be running before you run deploy.sh.
03

Install k3d + kubectl

k3d runs a full Kubernetes cluster inside Docker containers. kubectl is the CLI.

Terminal
bash
$ brew install k3d kubectl
$ k3d version          # → k3d version v5.x.x
$ kubectl version --client # → Client Version: v1.28.x
macOS ready checklist
  • docker ps returns an empty table (no error)
  • k3d version returns v5.x or higher
  • kubectl version --client returns a version number

Run PowerShell as Administrator where noted.

WSL2 is required for Docker Desktop's Linux container support. One-time setup, ~5 minutes.
01

Enable WSL2

Open PowerShell as Administrator (right-click Start → Windows PowerShell (Admin)).

PowerShell (Administrator)
powershell
# Installs WSL2 with Ubuntu — requires a restart
PS> wsl --install
# After restart:
PS> wsl --set-default-version 2
PS> wsl --list --verbose   # Ubuntu should show VERSION 2
After the restart, Ubuntu will ask you to create a Linux username and password. Use something simple — you will need it for sudo inside WSL.
02

Install Docker Desktop

PowerShell
powershell
PS> winget install Docker.DockerDesktop
# Open Docker Desktop after install
# Settings → General → ✓ "Use WSL 2 based engine"
# Settings → Resources → WSL Integration → Enable for Ubuntu
PS> docker --version   # → Docker version 26.x.x
Docker Desktop must be running before you run the deploy script.
03

Install k3d + kubectl

Option A: native Windows via winget. Option B: inside WSL2 (recommended for bash deploy.sh).

PowerShell — Option A (native)
powershell
PS> winget install k3d
PS> winget install Kubernetes.kubectl
# Reopen PowerShell, then:
PS> k3d version   # → k3d version v5.x.x
WSL2 Ubuntu — Option B (for deploy.sh)
bash
$ curl -s https://raw.githubusercontent.com/k3d-io/k3d/main/install.sh | bash
$ sudo apt-get install -y kubectl
$ k3d version   # → k3d version v5.x.x
04

Allow PowerShell scripts (if using deploy.ps1)

PowerShell (Administrator)
powershell
PS> Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
02

Get the Code + Zscaler Certificate

01

Clone the repository

Terminal
bash
$ git clone https://github.com/MegaDragon-Lab/PerfStack.git
$ cd PerfStack
$ ls deploy.sh   # confirm it's there
PowerShell or WSL2
bash
$ git clone https://github.com/MegaDragon-Lab/PerfStack.git
$ cd PerfStack
# In WSL2, Windows drives are at /mnt/c/Users/YourName/...
02

Copy the Zscaler certificate

Required on FICO network. The deploy script uses it when pulling images and building containers.

Terminal
bash
# Copy the cert into the project root — exact filename matters
$ cp /path/to/ZscalerRootCA.pem ./zscaler.pem

# Verify it looks right
$ head -1 zscaler.pem
-----BEGIN CERTIFICATE-----
How to export from Windows
steps
# Open certmgr.msc
#   → Trusted Root Certification Authorities → Certificates
#   → find "Zscaler Root CA"
#   → Right-click → All Tasks → Export...
#   → Base-64 encoded X.509 (.CER)
#   → Save as  zscaler.pem  in the PerfStack project folder
If zscaler.pem is missing the script will warn and continue, but Docker pulls may fail on a Zscaler-intercepted network. The cert is never committed to the repo.
03

Deploy — One Command

The script creates the k3d cluster, local registry, pre-pulls all infra images, builds your app images, pushes them to the registry, applies all Kubernetes manifests, and waits for every pod to be healthy.

02

Re-deploy after code changes

The script is fully idempotent — skips existing cluster and registry, rebuilds only the app images.

Terminal
bash
# After editing backend/ or frontend/ source:
$ ./deploy.sh

# Full teardown and fresh start:
$ k3d cluster delete perfstack && ./deploy.sh

05

Troubleshooting

!

deploy.sh fails — "Docker is not running"

fix
bash
# Start Docker Desktop, wait for whale icon to be steady, then:
$ ./deploy.sh
fix
powershell
# Start Docker Desktop, wait for it to be fully running, then:
PS> .\deploy.ps1   # or: ./deploy.sh from WSL2
!

Pods stuck in ImagePullBackOff

k3s nodes could not pull from the local registry. Usually means the cluster was created without the registry config. Full teardown fixes it.

fix
bash
$ k3d cluster delete perfstack
$ ./deploy.sh
!

docker pull fails — x509 / certificate error

Zscaler is intercepting HTTPS and the cert is missing or untrusted.

fix
bash
# Verify zscaler.pem is in the project root
$ head -1 zscaler.pem
-----BEGIN CERTIFICATE-----

# Also add to Docker Desktop: Settings → Docker Engine → add
# the cert under "insecure-registries" or trust via system keychain
!

Port 80 already in use

fix
bash
# Find what is using port 80
$ sudo lsof -i :80
# Stop the conflicting process, then:
$ k3d cluster delete perfstack && ./deploy.sh
fix — PowerShell
powershell
# Find what is using port 80
PS> netstat -ano | findstr :80
# Stop the process, then re-run deploy
!

Ask Claude Code to debug the cluster

Claude Code prompts
prompts
"Run kubectl get pods -n perfstack and kubectl describe pod
<pod-name> -n perfstack for any pod not in Running state.
Tell me what is wrong and how to fix it."

"Run kubectl logs deployment/backend -n perfstack --tail=50
and identify any startup errors."
This is a great live demo of Claude Code — it reads actual cluster state and proposes a specific fix rather than generic advice.
