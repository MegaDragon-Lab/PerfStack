Post-Workshop · Local Setup Guide

PerfStack
on your laptop

This guide walks you through installing PerfStack locally on your own machine — Windows or macOS. One script does everything: creates the k3d cluster, builds both Docker images, pushes them to a local registry, and deploys all services. No cloud dependency, no tunnel, no admin rights needed.

01

Required Tools

Only three tools are needed on your laptop. Everything else (Python, Node.js, k6, InfluxDB, Grafana) runs inside the cluster — the deploy script handles it automatically.

Tool Purpose macOS Windows Version
Docker Desktop Container runtime (host + cluster) Homebrew Winget 4.x+
k3d k3s Kubernetes cluster in Docker Homebrew Winget / WSL2 5.x+
kubectl Kubernetes CLI Homebrew Winget 1.28+
Git Clone the repo Xcode / Homebrew git-scm.com any
zscaler.pem Corporate proxy cert (FICO network) Copy manually to project root —
k3d runs k3s (a lightweight Kubernetes distribution) entirely inside Docker containers, so it uses your existing Docker Desktop — no separate VM or hypervisor needed. The deploy script creates the cluster, a local image registry, builds your images, and deploys everything. No manual kubectl commands required.
02

macOS Setup

We use Homebrew as the package manager. All commands run in Terminal.app or iTerm2.

01

Install Homebrew

Skip if already installed.

Terminal
bash
$ /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Verify
$ brew --version
Homebrew 4.x.x
On Apple Silicon (M1/M2/M3/M4) add Homebrew to PATH after install: echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile && source ~/.zprofile
02

Install Docker Desktop

Docker Desktop provides the container runtime that k3d uses. Must be running before deploy.

Terminal
bash
$ brew install --cask docker

# Open Docker Desktop from Applications and complete the setup wizard
# Wait until the Docker icon in the menu bar is steady (not animated)

# Verify
$ docker --version
Docker version 26.x.x
Docker Desktop must be running before you run deploy.sh. Check the whale icon in the menu bar.
03

Install k3d + kubectl

k3d creates a full Kubernetes cluster inside Docker containers. kubectl is the CLI to interact with it.

Terminal
bash
$ brew install k3d kubectl

# Verify
$ k3d version
k3d version v5.x.x

$ kubectl version --client
Client Version: v1.28.x
macOS ready checklist
  • docker ps returns an empty table (no error)
  • k3d version returns v5.x or higher
  • kubectl version --client returns a version number
03

Windows Setup

We use WSL2 + winget. Run PowerShell commands as Administrator where noted. The deploy script runs from either WSL2 (bash) or native PowerShell.

Windows requires WSL2 for Docker Desktop's Linux container support. This is a one-time setup (~5 min).
01

Enable WSL2

Open PowerShell as Administrator (right-click Start → Windows PowerShell (Admin)).

PowerShell (Administrator)
powershell
# Install WSL2 with Ubuntu — requires a restart
PS> wsl --install

# After restart, set WSL2 as default
PS> wsl --set-default-version 2
PS> wsl --list --verbose
  NAME      STATE   VERSION
* Ubuntu    Running 2
After the restart Windows will prompt you to create a Linux username and password for Ubuntu. Use something simple — you will need it for sudo inside WSL.
02

Install Docker Desktop

Uses the WSL2 backend automatically.

PowerShell
powershell
PS> winget install Docker.DockerDesktop

# Open Docker Desktop after install
# Settings → General → ✓ "Use WSL 2 based engine"
# Settings → Resources → WSL Integration → Enable for Ubuntu

PS> docker --version
Docker version 26.x.x
03

Install k3d + kubectl

Install natively via winget, or inside WSL2 via the bash script. Both work.

PowerShell — native Windows
powershell
PS> winget install k3d
PS> winget install Kubernetes.kubectl

# Reopen PowerShell, then verify
PS> k3d version
k3d version v5.x.x
WSL2 Ubuntu — alternative
bash
# Inside WSL2 terminal
$ curl -s https://raw.githubusercontent.com/k3d-io/k3d/main/install.sh | bash
$ sudo apt-get install -y kubectl
Running the bash deploy.sh inside WSL2 is the easiest path on Windows — Docker Desktop exposes its socket to WSL2 automatically.
04

Allow PowerShell scripts (if using deploy.ps1)

Windows blocks unsigned scripts by default. Run once as Administrator.

PowerShell (Administrator)
powershell
PS> Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
04

Get the Code + Zscaler Certificate

01

Clone the PerfStack repository

Terminal / WSL2 / PowerShell
bash
$ git clone https://github.com/your-org/perfstack.git
$ cd perfstack

# Confirm the deploy script is present
$ ls deploy.sh deploy.ps1
deploy.sh   deploy.ps1
02

Copy the Zscaler certificate into the project root

Required on any machine behind the FICO corporate proxy. The deploy script uses it when pulling images and building Docker images.

macOS / Linux
bash
# Copy the cert into the project root — exact filename matters
$ cp /path/to/ZscalerRootCA.pem ./zscaler.pem

# Verify it is a PEM file
$ head -1 zscaler.pem
-----BEGIN CERTIFICATE-----
Windows — where to export the cert
text
# Open certmgr.msc  →  Trusted Root Certification Authorities
#   → Certificates → find "Zscaler Root CA"
#   → Right-click → All Tasks → Export...
#   → Base-64 encoded X.509 (.CER)
#   → Save as zscaler.pem in the project root
If zscaler.pem is missing the deploy script will warn and continue, but Docker image pulls may fail on a Zscaler-intercepted network. The cert is never committed to the repo.
05

Deploy — One Command

A single script does everything: creates the k3d cluster, sets up a local image registry, pre-pulls all infra images, builds the backend and frontend Docker images, pushes them to the registry, applies all Kubernetes manifests, and waits for every pod to be healthy.

01

macOS / Linux

Make sure Docker Desktop is running, then:

02

Windows — Option A: WSL2 (recommended)

Open the WSL2 Ubuntu terminal, navigate to the project, run the bash script.

WSL2 Ubuntu Terminal
bash
# Navigate to the cloned repo (Windows paths accessible via /mnt/c/...)
$ cd /mnt/c/Users/YourName/perfstack
$ chmod +x deploy.sh && ./deploy.sh
03

Windows — Option B: Native PowerShell

PowerShell
powershell
PS> cd C:\Users\YourName\perfstack
PS> .\deploy.ps1
04

Re-deploy after code changes

The script is fully idempotent — it skips the cluster and registry if they already exist, rebuilds only the app images, and restarts affected deployments.

Terminal
bash
# Edit backend/ or frontend/ source code, then:
$ ./deploy.sh

# Full teardown and fresh start:
$ k3d cluster delete perfstack && ./deploy.sh

07

Troubleshooting

!

deploy.sh fails — "Docker is not running"

fix
bash
# Start Docker Desktop, wait for the whale icon to be steady, then re-run:
$ ./deploy.sh
!

Pods stuck in ImagePullBackOff

The k3s nodes could not pull an image from the local registry. Usually means the cluster was created without the registry config.

fix
bash
# Full teardown and fresh deploy (takes ~5 min)
$ k3d cluster delete perfstack
$ ./deploy.sh
!

docker pull fails — x509 / certificate error

The Zscaler proxy is intercepting HTTPS traffic and your cert is missing or not trusted.

fix
bash
# Verify zscaler.pem is present in the project root
$ head -1 zscaler.pem
-----BEGIN CERTIFICATE-----

# Add zscaler.pem to Docker Desktop's trusted certs:
# Docker Desktop → Settings → Docker Engine → add to "registry-mirrors"
# or import via macOS Keychain / Windows certmgr.msc and restart Docker Desktop
!

Port 80 already in use

Another process (nginx, Apache, another k3d cluster) is bound to port 80.

fix
bash
# Find what is on port 80
$ sudo lsof -i :80

# Delete any existing perfstack cluster first
$ k3d cluster delete perfstack

# Stop the conflicting process, then re-run deploy.sh
!

Ask Claude Code to debug the cluster

Claude Code prompts
prompts
"Run kubectl get pods -n perfstack and kubectl describe pod
<pod-name> -n perfstack for any pod not in Running state.
Tell me what is wrong and how to fix it."

"Run kubectl logs deployment/backend -n perfstack --tail=50
and identify any startup errors."

"Run kubectl get events -n perfstack --sort-by=.lastTimestamp
and summarise what went wrong."
This is a great live demo of Claude Code — it reads actual cluster state, understands the error context, and proposes a specific fix rather than generic advice.
