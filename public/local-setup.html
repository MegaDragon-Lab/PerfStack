PerfStack
on your laptop

This guide walks you through installing PerfStack locally on your own machine — Windows or macOS. Once set up, you have a permanent local Kubernetes environment to keep building with Claude Code, no cloud dependency needed.

Required Tools

These are the tools you need on your laptop. The table below shows the recommended install method for each platform — sections 02 and 03 walk through them step by step.

Tool Purpose macOS Windows Version
Docker Desktop Container runtime for Minikube Homebrew Winget 4.x+
Minikube Local Kubernetes cluster Homebrew Winget 1.32+
kubectl K8s CLI Homebrew Winget 1.28+
Node.js 18+ Required by Claude Code Homebrew Winget 18 LTS

Your machine needs at least 8 GB RAM and 6 GB free disk space. Minikube will use 4 GB RAM and 2 CPUs by default. On a 16 GB machine it runs comfortably alongside a browser and your editor.
02

macOS Setup

We use Homebrew as the package manager. If you already have Homebrew, skip step 1. All commands run in Terminal.app or iTerm2.

01

Install Homebrew

The missing package manager for macOS. Skip if already installed.

Terminal
bash
$ /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Verify
$ brew --version
Homebrew 4.x.x
On Apple Silicon (M1/M2/M3) Homebrew installs to /opt/homebrew. Add it to your PATH if the shell doesn't find brew after install: echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
02

Install Docker Desktop

Docker Desktop provides the container runtime that Minikube uses as its driver on macOS.

Terminal
bash
$ brew install --cask docker

# Open Docker Desktop from Applications and complete the setup wizard
# Wait until the Docker icon in the menu bar shows "Docker Desktop is running"

# Verify
$ docker --version
Docker version 24.x.x
Docker Desktop must be running before you start Minikube. Check the whale icon in the menu bar — it must be steady, not animated.
03

Install Minikube + kubectl

Terminal
bash
$ brew install minikube kubectl

# Verify both
$ minikube version
minikube version: v1.32.x

$ kubectl version --client
Client Version: v1.28.x
04

Install Node.js, Python 3.11 and Git

Terminal
bash
$ brew install node@18 [email protected] git

# Link node 18 if you have multiple versions
$ brew link node@18 --force --overwrite

# Verify
$ node --version   # → v18.x.x
$ python3 --version # → Python 3.11.x
05

Install Claude Code

Terminal
bash
$ npm install -g @anthropic-ai/claude-code

# Verify
$ claude --version
claude 1.x.x

# Set your Anthropic API key (add to ~/.zshrc or ~/.bash_profile to persist)
$ export ANTHROPIC_API_KEY=sk-ant-...
To make the API key permanent on macOS, add the export line to your ~/.zshrc file and run source ~/.zshrc.
03

Windows Setup

We use winget (built into Windows 10/11) and WSL2 for the best experience. All commands run in PowerShell (run as Administrator where noted) or the WSL2 terminal.

Windows requires WSL2 (Windows Subsystem for Linux 2) so Docker Desktop can run Linux containers properly. This is a one-time setup that takes about 5 minutes.
01

Enable WSL2

Open PowerShell as Administrator (right-click Start → Windows PowerShell (Admin)).

PowerShell (Administrator)
powershell
# Install WSL2 with Ubuntu — this will require a restart
PS> wsl --install

# After restart, verify WSL2 is the default
PS> wsl --set-default-version 2
PS> wsl --list --verbose
  NAME      STATE   VERSION
* Ubuntu    Running 2
After the restart, Windows will open Ubuntu and ask you to create a Linux username and password. Set something simple — you'll use it for sudo commands inside WSL.
02

Install Docker Desktop

Docker Desktop for Windows uses the WSL2 backend automatically.

PowerShell
powershell
PS> winget install Docker.DockerDesktop

# After install, open Docker Desktop
# Settings → General → ✓ "Use WSL 2 based engine" (should be checked by default)
# Settings → Resources → WSL Integration → Enable for Ubuntu

# Restart Docker Desktop, then verify from PowerShell
PS> docker --version
Docker version 24.x.x
03

Install Minikube + kubectl

PowerShell
powershell
PS> winget install Kubernetes.minikube
PS> winget install Kubernetes.kubectl

# Close and reopen PowerShell, then verify
PS> minikube version
minikube version: v1.32.x

PS> kubectl version --client
Client Version: v1.28.x
On Windows, Minikube uses the docker driver (which requires Docker Desktop running). Do not use the Hyper-V driver — it conflicts with WSL2.
04

Install Node.js 18 and Python 3.11

PowerShell
powershell
PS> winget install OpenJS.NodeJS.LTS
PS> winget install Python.Python.3.11

# Close and reopen PowerShell to refresh PATH, then verify
PS> node --version   # → v18.x.x
PS> python --version  # → Python 3.11.x
If python command is not found after install, look for it as py or python3 — Windows sometimes registers Python under those aliases.
05

Install Claude Code

PowerShell
powershell
PS> npm install -g @anthropic-ai/claude-code

# Verify

claude 1.x.x

# Set your API key (for the current session)

# Start → "Edit the system environment variables" → Environment Variables
# → New (User variables) → Name: ANTHROPIC_API_KEY · Value: sk-ant-...

04

Start the Kubernetes Cluster

Same commands on both platforms. Make sure Docker Desktop is running before executing these.

01

Start Minikube

Allocate at least 4 GB RAM and 2 CPUs. First start downloads the base image (~600 MB).

Terminal / PowerShell
bash
# Start with Docker driver (works on both Mac and Windows)
$ minikube start --driver=docker --memory=4096 --cpus=2

😄  minikube v1.32.x
✨  Using the docker driver
🔥  Creating docker container (CPUs=2, Memory=4096MB)
🏄  Done! kubectl is now configured to use "minikube"
On a machine with 16 GB+ RAM, use --memory=6144 --cpus=4 for better performance with all PerfStack pods running simultaneously.
02

Verify the cluster

Terminal / PowerShell
bash
$ kubectl get nodes
NAME       STATUS   ROLES           AGE   VERSION
minikube   Ready    control-plane   30s   v1.28.x

$ minikube status
minikube: Running
kubelet:  Running
apiserver: Running

# Point Docker CLI to Minikube's daemon
# macOS / Linux:
$ eval $(minikube docker-env)
# Windows PowerShell:
PS> minikube docker-env | Invoke-Expression
The eval $(minikube docker-env) command only applies to your current terminal session. You need to run it every time you open a new terminal before building Docker images.
Environment ready checklist
  • docker ps returns an empty table (no error)
  • minikube status shows Running on all three lines
  • kubectl get nodes shows minikube with STATUS=Ready
  • claude --version returns a version number
  • ANTHROPIC_API_KEY is set in the environment
05

Build PerfStack with Claude Code

With the cluster running, you can now reproduce everything from the workshop on your own machine. Start Claude Code in your project directory and follow the lab steps from the workshop guide.

01

Clone the workshop repo (if shared) or start fresh

Terminal / PowerShell
bash
# Option A — clone the workshop repo
$ git clone https://github.com/your-org/perfstack.git
$ cd perfstack

# Option B — start from scratch (recommended: build it yourself!)
$ mkdir perfstack && cd perfstack
$ claude
02

Run the deploy script

Build and deploy all services to your local Minikube cluster.

macOS / Linux
bash
# Make sure Docker is pointing at Minikube first
$ eval $(minikube docker-env)
$ chmod +x deploy.sh && ./deploy.sh
Windows PowerShell
powershell
# Point Docker at Minikube
PS> minikube docker-env | Invoke-Expression

# Build images manually (deploy.sh is a bash script)
PS> docker build -t perfstack-backend:latest ./backend
PS> docker build -t perfstack-frontend:latest ./frontend
PS> kubectl apply -f k8s/
PS> kubectl rollout status deployment/backend -n perfstack
PS> kubectl rollout status deployment/frontend -n perfstack
Windows users: ask Claude Code to generate a deploy.ps1 PowerShell script equivalent to the bash deploy.sh — it will handle the syntax differences automatically.
03

Open the services in your browser

Terminal / PowerShell
bash
# Get Minikube IP
$ minikube ip
192.168.49.2

# Or open services directly in the browser (minikube handles the tunnel)
$ minikube service frontend -n perfstack
$ minikube service grafana  -n perfstack

# All services at once
$ minikube service list -n perfstack
|-----------|----------|-------------|--------------------------|
| NAMESPACE |   NAME   | TARGET PORT |            URL           |
|-----------|----------|-------------|--------------------------|
| perfstack | frontend |        3000 | http://192.168.49.2:30080|
| perfstack | grafana  |        3000 | http://192.168.49.2:30300|
| perfstack | backend  |        8000 | http://192.168.49.2:30800|
|-----------|----------|-------------|--------------------------|

07

Troubleshooting

!

Minikube won't start — "Docker is not running"

Docker Desktop must be running before minikube start. Check the system tray / menu bar icon.

fix
bash
# Start Docker Desktop, wait for it to be fully running, then:
$ minikube delete && minikube start --driver=docker --memory=4096 --cpus=2
!

Pods stuck in ImagePullBackOff

The Docker images were built on the host daemon, not Minikube's. You forgot eval $(minikube docker-env) before building.

fix
bash
# macOS/Linux
$ eval $(minikube docker-env)
$ docker build -t perfstack-backend:latest ./backend
$ docker build -t perfstack-frontend:latest ./frontend
$ kubectl rollout restart deployment/backend deployment/frontend -n perfstack

# Windows PowerShell
PS> minikube docker-env | Invoke-Expression
PS> docker build -t perfstack-backend:latest ./backend
!

Windows: "execution of scripts is disabled on this system"

PowerShell script execution policy is restricted by default on Windows.

PowerShell (Administrator)
powershell
# Allow local scripts (safe — only affects local, unsigned scripts)
PS> Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
!

Not enough memory — pods in Pending state

Minikube doesn't have enough RAM allocated. Delete and recreate with more memory.

fix
bash
$ minikube stop
$ minikube delete
$ minikube start --driver=docker --memory=6144 --cpus=4
You need at least 8 GB total RAM on your machine to allocate 6 GB to Minikube. Close other memory-heavy apps (browsers with many tabs, other VMs) before starting.
!

Ask Claude Code to debug

When something is broken, Claude Code can inspect the cluster state and propose a fix.

Claude Code prompts for debugging
prompts
"Run kubectl get pods -n perfstack and kubectl describe pod
 <pod-name> -n perfstack for any pod not in Running state.
 Tell me what is wrong and how to fix it."

"Run kubectl logs deployment/backend -n perfstack --tail=50
 and identify any startup errors."

"Run kubectl get events -n perfstack --sort-by=.lastTimestamp
 and summarise what went wrong."
This is a great live demo of Claude Code's power — it reads the actual cluster state, understands the error context, and proposes a specific fix rather than generic advice.
