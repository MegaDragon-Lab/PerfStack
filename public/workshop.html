02

IAM authentication module

OAuth2 client credentials flow with automatic token caching and renewal.

"Create backend/auth.py with a Python dataclass IamAuthClient that accepts iam_url, client_id and client_secret. Implement async get_bearer_token() — it must POST to iam_url with Content-Type application/x-www-form-urlencoded and body: grant_type=client_credentials, client_id, client_secret. Cache the token internally and refresh 30 seconds before expiry. Use structured logging. Never log the secret value or the token itself."

backend/auth.py — Claude Code output
python
@dataclass
class IamAuthClient:
    iam_url: str
    client_id: str
    client_secret: str
    _token: str    = field(default="", repr=False)
    _expires_at: float = field(default=0.0, repr=False)

    async def get_bearer_token(self) -> str:
        if self._token and time.time() < self._expires_at - 30:
            return self._token     # cached ✓
        return await self._fetch_token()
    # ... Claude Code completes the full implementation
Follow up: "Add pytest unit tests for auth.py in backend/tests/ using pytest-asyncio and respx to mock HTTP calls. Include a test for the cache hit and one for token refresh."
03

FastAPI backend — orchestration API

The backend ties IAM auth, K6 script generation and Kubernetes job lifecycle together.

"Create backend/main.py with FastAPI. Endpoints: POST /api/run-test accepting {iam_url, client_id, client_secret, target_url, payload, vus, duration} — fetch Bearer token via IamAuthClient, render a K6 script via Jinja2 and create a Kubernetes Job. GET /api/test-status/{job_name} returns {status, message}. Add CORS for localhost:3000, a /health endpoint, and Pydantic models for all request/response bodies."

backend/main.py — POST /api/run-test
python
@app.post("/api/run-test")
async def run_test(config: TestConfig):
    # 1. Bearer token from IAM
    auth  = IamAuthClient(config.iam_url, config.client_id, config.client_secret)
    token = await auth.get_bearer_token()

    # 2. Render K6 script template with live token + config
    script = render_k6_script(token, config)

    # 3. Launch Kubernetes Job
    job_name = await create_k6_job(script, config)
    return {"job_name": job_name, "status": "created"}
04

Dynamic K6 script with Bearer auth

A Jinja2 template rendered per test run — Bearer token, URL and payload injected at runtime.

"Create backend/k6_template.js as a Jinja2 template. It must: inject Bearer token in Authorization header, POST to {{ target_url }} with {{ payload | tojson }}, use ramp-up stages with {{ vus }} VUs and {{ duration }}s, set thresholds p95 < 2000ms and error rate < 5%, output metrics to InfluxDB at http://influxdb:8086/k6."

backend/k6_template.js
javascript
import http  from 'k6/http';
import { check, sleep } from 'k6';

export const options = {
  stages: [
    { duration: '30s',          target: {{ vus }} },
    { duration: '{{ duration }}s', target: {{ vus }} },
    { duration: '10s',          target: 0 },
  ],
  thresholds: {
    'http_req_duration': ['p(95)<2000'],
    'http_req_failed':   ['rate<0.05'],
  },
};

export default function() {
  const res = http.post('{{ target_url }}', JSON.stringify({{ payload }}), {
    headers: {
      'Authorization': 'Bearer {{ bearer_token }}',
      'Content-Type':  'application/json',
    }
  });
  check(res, { 'status 2xx': r => r.status >= 200 && r.status < 300 });
  sleep(1);
}

Dark panel to configure and fire tests. Live status polling via the backend.

"Create frontend/src/App.jsx. Build a dark config panel with fields: IAM URL, Client ID, Client Secret (password input), Target URL, JSON Payload (textarea), VUs (range slider 1–500 with live value label), Duration in seconds (range slider 10–600). A 'Run Load Test' button calls POST /api/run-test. Poll GET /api/test-status/{job_name} every 3 seconds and show a status badge. When status is 'running', show a link to Grafana at port 30300. Validate JSON payload client-side before submitting. Use Tailwind CSS."

Follow up: "Add a collapsible 'Advanced' section with fields for custom K6 thresholds (p95 target ms, max error rate %) that get passed to the backend."
06

Kubernetes manifests + Grafana dashboard

All K8s resources: namespace, deployments, services, RBAC and a pre-provisioned Grafana dashboard.

"Generate all files under k8s/ in namespace 'perfstack'. Create: namespace.yaml, Deployments + Services for backend (NodePort 30800), frontend (NodePort 30080), InfluxDB (ClusterIP 8086), Grafana (NodePort 30300). Add a ServiceAccount 'k6-runner-sa' with a Role allowing create/get/list/watch on batch/jobs and get/list on pods. Add ConfigMaps for Grafana: one datasource pointing to InfluxDB db=k6, one dashboard JSON showing RPS, active VUs, p50/p95/p99 latency and error rate."

The backend pod uses python-kubernetes to create Jobs — it needs a ServiceAccount bound to a Role with batch permissions. Claude Code will generate the full RBAC chain automatically.
